# Definitions

## Purpose

This directory defines the entities and concepts that exist within the game world.

Its purpose is to establish a shared understanding of each concept before any game mechanics, content, or scenarios reference it.

Definitions describe what something is, not how it behaves.

---

# Scope

A definition may describe:

* The identity of an entity.
* Its purpose within the game world.
* Its possible attributes.
* Its relationships with other entities.
* Notes that help clarify interpretation.

A definition should remain independent of any specific Game State.

---

# What Definitions Do Not Include

Definitions do not describe game mechanics.

Definitions do not specify calculations, probabilities, costs, or outcomes.

Definitions do not introduce game content.

Definitions do not describe scenario-specific information.

If information explains how something behaves during gameplay, it belongs in the Rules.

If information describes a specific object in the game world, it belongs in the Content.

If information establishes an initial Game State, it belongs in a Scenario.

---

# Relationships with Other Documentation

Definitions provide the foundation for the rest of the Game Documentation.

Rules describe how defined entities behave.

Content provides concrete instances of defined entities.

Scenarios establish an initial Game State using existing Definitions, Rules, and Content.

---

# Consistency

Each concept should be defined only once.

Other documents should reference an existing definition instead of redefining it.

Definitions should remain consistent with the canonical terminology established in the Glossary.

---

# Independence

Definitions should not depend on individual companies, people, models, or events.

They describe general concepts that may be reused throughout the Game Documentation.

---

# Examples

The following are examples of concepts that belong in this directory.

* Player
* Company
* Employee
* Model
* Technology
* GPU
* Dataset
* Event

These examples illustrate the types of entities that may be defined here. They do not represent the complete set of possible definitions.
